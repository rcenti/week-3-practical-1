Instructions

The goal of this exercise is to familiarise yourself with using the browser console to identify bugs and errors in your code.

The code in exercise1-1/sketch.js is supposed to produce the output shown in the video below. Run the code in your browser and you will see that it does not! The sketch stops after only one circle is drawn.

The code contains two bugs--one is an error that causes the program to crash and the other is a logic mistake that does not cause a crash but also does not produce the right output.

When your code appears to crash (e.g. no shapes are drawn even though the code is present), open the browser console by going to the View menu (in Chrome), selecting Developer, then choosing JavaScript Console. If there is an error message, it will help you identify what and where the problem is.

Fix the error in the code by opening the Console and reading the error message. To the right of the code, you should see a message that says "sketch.js:" followed by a number. This tells you that the error occurred in your sketch.js file and the number tells you the line number that the error is likely on. Use this information to help you fix the error.

When the error is fixed, you will see that only one row of circles is drawn. There is no error message this time but the code is misbehaving. This suggests a logic error, likely with one of the variables used to position the circles. Try using the p5.js print() function or JavaScript's console.log() to print the values of circleX and circleY at the end of draw(). You will need to work out what circleX and circleY should be as the sketch progresses in order to make use of the messages printed to the console.

The bug is very small and you shouldn't need to make big changes to the code!

When you are done, check your work using check-exercise1-1.html.

Expected output:

The error that causes the program to crash is a typo in a variable name. The variable name and line number are given in the error message.

The logic error is harder to identify. Here is what to look for when you print the values of circleX and circleY:

  • circleX should start at half the width of the circle and increase by the width of the circle until the circles reach the right edge. Then, because of the % operator, circleX should return to its original value before increasing again as before.
  • circleY should start at half the width of the circle and stay the same until the circles reach the right edge of the screen. At this point, circleY should increase by the width of the circle, starting a new row. This pattern should continue until the circles reach the bottom edge. Then, circleY should return to its original value.

The error is caused by the order of operations in a calculation (think BODMAS or PEMDAS!). The modulo operator has the same precedence as multiplication or division.
